# MLForge v2 — Сценарий использования

## Обзор

MLForge v2 — это локальный инструмент для полного цикла работы с проектами компьютерного зрения (CV): от создания датасета и разметки до обучения модели и анализа экспериментов.

---

## Запуск приложения

```bash
# Через Docker (рекомендуется)
docker-compose up --build

# Или вручную
# Backend
cd backend && pip install -e . && uvicorn app.main:app --host 0.0.0.0 --port 8001

# Frontend
cd frontend && npm install && npm run dev
```

Приложение будет доступно на `http://localhost:8001` (Docker) или `http://localhost:5173` (dev-режим).

## Рабочий процесс: от датасета до модели

### Шаг 1: Dashboard (Главная)

**Что это:** Обзорная панель с ключевыми метриками всего воркспейса.

**Что показывает:**
- Количество проектов, датасетов, обученных моделей и экспериментов
- Графики активности и прогресса разметки
- Последние действия

**Как использовать:** Используйте как стартовую точку для навигации. Здесь видно общее состояние работы.

---

### Шаг 2: Projects (Проекты)

**Что это:** Организация работы по проектам.

**Как использовать:**
1. Нажмите **"New Project"** для создания проекта
2. Укажите имя и описание проекта
3. Проект объединяет связанные датасеты и эксперименты

---

### Шаг 3: Datasets (Датасеты)

**Что это:** Управление наборами данных для обучения.

**Как использовать:**
1. Создайте новый датасет — укажите имя, тип задачи (Object Detection, Classification), формат (YOLO, COCO)
2. **Scan Directory** — укажите путь к папке с изображениями, система автоматически найдёт и добавит файлы
3. **Extract Frames** — извлечение кадров из видеофайлов
4. Управляйте классами (метками): добавляйте имена классов и цвета для визуализации
5. Добавляйте теги для фильтрации изображений
6. Просматривайте статистику: количество изображений, классов, аннотаций

---

### Шаг 4: Labeling (Разметка)

**Что это:** Встроенный инструмент разметки изображений (аналог LabelImg/CVAT).

**Как использовать:**
1. Выберите датасет из списка и нажмите **"Start"** / **"Continue"**
2. Откроется редактор с тремя панелями:
   - **Левая** — список файлов с фильтрами (All / Unlabeled / Annotated / Awaiting review)
   - **Центр** — холст для рисования bounding box'ов
   - **Правая** — классы, объекты на изображении, теги, статус ревью

**Инструменты:**
- **Box** — рисование прямоугольных рамок (выберите класс → перетащите на изображении)
- **Pan** — перемещение по изображению (также через среднюю кнопку мыши)
- **Zoom** — масштабирование (колесо мыши или кнопки +/-)
- **Undo/Redo** — отмена/повтор действий (Ctrl+Z / Ctrl+Shift+Z)
- **Save** — сохранение аннотаций (автосохранение через 1 сек)
- **Save & Next** — сохранить и перейти к следующему изображению
- **Mark Verified** — отметить изображение как проверенное
- **Copy/Paste/Duplicate** — работа с выделенными объектами
- Стрелки ← → — навигация между изображениями

**Навигация:**
- **Label queue** — очередь неразмеченных изображений
- **Review queue** — очередь на проверку
- **Next unlabeled / Next review** — быстрый переход

---

### Шаг 5: AutoLabel (Авторазметка)

**Что это:** Автоматическая разметка с помощью предобученной модели.

**Как использовать:**
1. Выберите датасет для авторазметки
2. Выберите модель из зарегистрированных (или предобученную)
3. Настройте параметры: порог уверенности (confidence threshold), скорость обработки
4. Система покажет расчётное время и количество изображений по сплитам (train/validation/test)
5. Нажмите **"Start Auto-Labeling"** для запуска
6. После завершения перейдите в Labeling для проверки и корректировки результатов

---

### Шаг 6: Training (Обучение)

**Что это:** Запуск и мониторинг обучения моделей.

**Как использовать:**
1. Нажмите **"New Training"** для создания нового запуска
2. Укажите параметры: модель (YOLOv8n/s/m/l), количество эпох, learning rate, batch size
3. Таблица показывает все запуски с метриками:
   - **Name** — имя запуска
   - **Model** — архитектура модели
   - **Epochs** — текущая/всего эпох
   - **Best mAP@0.5** — лучшая метрика качества
   - **Precision** — точность
   - **Status** — статус (Running/Completed/Failed)
   - **Duration** — длительность
4. Графики отображают динамику потерь (loss) и метрик по эпохам
5. Используйте кнопку удаления (корзина) для очистки ненужных запусков

---

### Шаг 7: AutoML (Автоподбор)

**Что это:** Автоматический поиск лучших гиперпараметров.

**Как использовать:**
1. Нажмите **"Start AutoML Run"** для запуска серии экспериментов
2. Таблица **Top Runs** — ранжирование всех запусков по mAP@0.5 (от лучшего к худшему)
3. Боковая панель показывает 5 последних запусков
4. KPI-карточки:
   - **Total Runs** — общее количество запусков
   - **Best mAP@0.5** — лучший результат
   - **Running** — количество активных запусков
   - **Search Space** — пространство поиска

---

### Шаг 8: Experiments (Эксперименты)

**Что это:** Единый реестр всех экспериментов с возможностью сравнения.

**Что показывает:**
- Все training runs как эксперименты в одной таблице
- Метрики: mAP@0.5, Precision, Recall для каждого эксперимента
- Статус (Completed / Running / Pending / Failed)
- Общая статистика: количество экспериментов, best mAP, total compute time

**Боковая панель:**
- Лучший эксперимент с детальной информацией (модель, эпохи, статус, дата, метрики)
- Графики mAP и Loss по эпохам (для лучшего запуска)

---

### Шаг 9: Models (Модели)

**Что это:** Реестр обученных моделей.

**Как использовать:**
1. Нажмите **"Register Model"** для добавления модели
2. Таблица показывает: имя, архитектуру, фреймворк, статус, метрики
3. Боковая панель с деталями первой модели
4. Действия: Download, Delete
5. Метрики: mAP@0.5, Precision, Recall

---

### Шаг 10: Versions (Версии)

**Что это:** Управление версиями датасетов.

**Что показывает:**
- Список версий с описанием изменений
- Статистику по версиям (количество изображений, аннотаций)
- Сравнение метрик между версиями
- Историю изменений (changelog)

---

### Шаг 11: Pipelines (Пайплайны)

**Что это:** Визуализация ML-пайплайна.

**Что показывает:**
- Шаги пайплайна: Dataset → Preprocessing → Augmentation → Training → Evaluation → Export → Deploy
- Статус каждого шага (completed, running, pending)
- Шаблоны пайплайнов
- История запусков

---

### Шаг 12: Settings (Настройки)

**Что это:** Конфигурация воркспейса.

**Разделы:**
- **General** — имя проекта, описание, язык
- **Storage** — пути к данным, моделям и логам
- **Training** — параметры по умолчанию (GPU, mixed precision, early stopping)
- **Toggles** — вкл/выкл функций (автосохранение, уведомления, метрики)
- **Integrations** — подключение внешних сервисов (W&B, MLflow, S3, etc.)
- **Danger Zone** — очистка кэша, экспорт/импорт, удаление воркспейса

Все настройки сохраняются в базу данных через API.

---

## Типовой рабочий процесс

```
1. Создать проект (Projects)
2. Создать датасет и загрузить изображения (Datasets → Scan Directory)
3. Добавить классы для разметки (Datasets → Classes)
4. Разметить изображения (Labeling)
5. [Опционально] Использовать автоматическую разметку (AutoLabel)
6. Запустить обучение модели (Training)
7. [Опционально] Запустить AutoML для подбора параметров (AutoML)
8. Сравнить эксперименты (Experiments)
9. Зарегистрировать лучшую модель (Models)
10. Создать версию датасета для воспроизводимости (Versions)
```

## Горячие клавиши (Labeling)

| Клавиша | Действие |
|---------|----------|
| ← / → | Предыдущее / следующее изображение |
| ↑ / ↓ | Предыдущая / следующая аннотация |
| Ctrl+Z | Отмена |
| Ctrl+Shift+Z | Повтор |
| Ctrl+C | Копировать выделенные объекты |
| Ctrl+V | Вставить объекты |
| Delete | Удалить выделенные объекты |
| Ctrl+D | Дублировать |
| Alt+Стрелки | Сдвинуть выделенный объект |
| V | Переключить режим Verified |
| F | Сбросить масштаб (Fit) |
| Колесо мыши | Зум |
| Средняя кнопка | Перемещение |
| Shift+Click | Мульти-выделение диапазона |
| Ctrl+Click | Добавить/убрать из выделения |
